## 1. What shipped

- Migration: `server/migrations/004-phase3-social-tables.sql` — `platform_accounts`, `posts`, `post_metrics` (time series, one row per capture), and `account_metrics` (follower snapshots), adapted from PropIQ's design doc to live conventions: integer ids, `propiq` schema, no platform CHECK, and no token storage in the DB. Applied live.
- Fetcher: `server/fetchers/instagram.js`, written fresh. PropIQ's `instagram.js` does not exist anywhere on this machine (only two SQL files in Downloads); there was nothing to port. The known fixes are built in: the `views` metric (`impressions` is deprecated), and media matched to stored URLs by permalink code — post ids are never derived from URLs. Supports both token flavors, auto-detected by prefix: `EAA...` (Facebook login, graph.facebook.com, needs IG_USER_ID) and `IG...` (Instagram business login, graph.instagram.com, `/me`).
- Sync: `POST /api/marketing/sync/instagram` — upserts the account, snapshots followers, matches the account's media against `platform_links`, stores posts + a metrics capture each, resolves `platform_links.post_id` placeholders with real API ids, and refreshes `analytics_cache`/`analytics_history` per touched property.
- Marketing page: account bar with Sync button, KPIs (followers, reach, engagement rate, posts tracked), follower trend chart, top-posts table.
- Lead capture: `POST /api/leads` and an Add-a-Lead form on the Sales page (name, phone, property, source, stage); every Sales widget refreshes on save. Closes Phase 2's biggest gap.
- `contractRoute` gained the `httpError` pattern: errors thrown with `expose` carry a user-readable message and status through to the client (used for "credentials not configured", Graph API failures, and 404s).
- Credential entry scripts: `scripts/add-instagram-keys.ps1` (prompting) and `scripts/token-from-clipboard.ps1` (reads the token straight from the clipboard — the only method that survived contact with this machine's paste behavior).
- Verified live: first sync pulled @bluefalcon_estateagents (306 followers, 653 media). The user then linked a real post and re-synced on their own: 1 post tracked, 120 reach.

## 3. Environment state

- **The Meta app changed.** The brief said to reuse the "PropIQ Sync" Meta app; that proved impossible — Meta locks an app's available use cases at creation, and PropIQ Sync was created Facebook-Login-only (its "Add use cases" dialog offers nothing relevant). A new app, **Blue Falcon Analytics** (app id 1564314185307543), was created with the Pages and Instagram use cases. PropIQ Sync is untouched and abandoned for analytics purposes.
- Instagram is connected via **Instagram business login** (Meta removed Instagram permissions from the Pages use case): the IG account `bluefalcon_estateagents` holds the Instagram Tester role on the app, and the token is generated by a dashboard button (Use cases → Manage messaging & content on Instagram → Customize → Generate token). These tokens are long-lived (~60 days).
- `.env` now holds `META_ACCESS_TOKEN` (the `IGAA...` token) and `IG_USER_ID=17841428304887753`. Declared in `.env.example` and validated (optionally) by `env.js`.
- Token renewal (~every 60 days): regenerate in the dashboard, click Copy, run `scripts/token-from-clipboard.ps1`, restart the dev server.
- Row state at close: 4 social tables live; one instagram account row; posts/metrics accumulating from real syncs.

## 4. Open issues / known-broken

- **Token expiry is a silent time bomb (~60 days).** When the sync starts failing with an auth error around early September 2026, the fix is the renewal flow above. Nothing warns proactively.
- **Only linked posts are tracked.** 653 media exist on the account; only posts linked to a listing on the Listings page get metrics. That is by design (property attribution is the product), but the numbers will look "small" next to the account's real activity.
- Instagram permalink matching covers `/p/`, `/reel/`, `/reels/`, `/tv/` URL forms. Anything else (e.g. share links with no code) counts as unmatched and is reported in the sync summary.
- The app is unpublished/in development mode — fine for the admin's own account, forever. Publishing (App Review, business verification) is only needed if other users' accounts should ever connect; avoid it.

## 5. Decisions and why

- **The fetcher was written fresh, not ported.** Correction to the brief: the PropIQ fetcher code does not exist on this machine. The brief's warnings about its bugs served as the spec for what to avoid.
- **Match by permalink code, never derive ids from URLs.** Phase 1 established stored shortcodes don't map to Graph API media ids. The sync lists the account's own media and matches codes both ways.
- **Both Meta token flavors are supported with auto-detection**, because Meta forced a mid-phase pivot from the Facebook-login route (permissions unobtainable on the old app) to Instagram business login. Supporting both cost little and keeps either path viable.
- **Secrets enter via committed scripts, clipboard-first.** Editor edits, chat paste, and hidden-prompt paste all failed on this machine (see memory and the scripts' comments). `Get-Clipboard` is the reliable channel; scripts validate shape before writing.
- **`account_metrics` is a new table** rather than extending `analytics_history` (which is property-scoped and has no followers column).
- Lead capture landed in this phase (recommended entry point from Phase 2) rather than waiting for a dedicated phase.
